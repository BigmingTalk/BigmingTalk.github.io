Unix共享文件原理及系统级I/O

计算机系统
  • IO
  • v-node表
  • 共享文件

I/O（输入/输出）是指在主存和外部设备（磁盘驱动器、终端和网络）之间拷贝数据的过程。所有的I/O设备都被模型化为文件，一个Unix文件就是一个m个字节序列，而所有输入输出都被当做对相应文件的读和写来执行。将设备映射为文件，允许Unix内核将所有输入和输出都能以一种统一且一致的方式来执行。

Unix I/O

常见的输入输出包括：

  • 打开文件：应用程序通过要求内核打开相应文件，来访问一个I/O设备。内核返回一个小的非负整数，叫做描述符，它在后续对此文件的所有操作中标识该文件。内核记录有关该打开文件的所有信息，而应用程序只需记住该描述符。Unix外壳为每个进程开始时都有三个打开文件：标准输入（描述符为0, STDIN_FILENO)标准输出（描述符为1, STDOUT_FILENO）和标准错误（描述符为2, STDERR_FILENO）。
  • 改变当前的文件位置：对于每个打开文件，内核保持一个文件位置k，初始为0，文件位置是从文件开始起始的字节偏移量。应用程序可以通过执行seek操作，显示地设置文件的当前位置为k。
  • 读写文件：一个读操作就是从文件拷贝n>0个字节到主存，从当前文件位置k开始，然后将k增加到k+n。给定一个大小为m字节的文件，当k>m时执行读操作会触发EOF（end-of-file）条件。文件结尾处没有明确的“EOF符号”。类似的写操作就是从主存拷贝n>0个字节到一个文件，从当前文件位置k开始，然后更新k。
  • 关闭文件：当应用完成了对文件的访问之后，就通知内核关闭该文件。作为响应，内核释放文件打开时创建的数据结构，并将该描述符恢复到可用的描述符池中。无路一个进程为何终止，内核都会关闭所有打开的文件并释放它们的存储器资源。

系统级I/O函数

open函数和close函数

open函数将filename转换为一个文件描述符，返回的描述符总是在进程中当前没有打开的最小描述符。

flags参数：

  • O_RDONLY：只读
  • O_WRONLY：只写
  • O_RDWR：可读可写

此外，flags参数也可以是一个或者更多位掩码的或，为写提供给一些额外的指示：

  • O_CREAT：如果文件不存在，则创建一个空文件
  • O_TRUNC：如果文件存在，就截断它
  • O_APPEND：每次写操作前，设置文件位置到文件的结尾处

例如fd = open("bigming.txt", O_WRONLY|O_APPEND, 0)指定在文件后面添加数据。

mode参数：

关闭一个已关闭的描述符会出错。

read函数和write函数

这里ssize_t是是有符号的int，而size_t是无符号的unsigned int。一个返回一个-1是的read最大值减少了一半，从4GB到2GB。

某些情况下，read和write传送的字节比应用程序要求的少，这些不足值表示有错误：

  • 读时遇到EOF
  • 从终端读文本行：如果打开文件是终端相关联的（如键盘和显示器），那么每个read函数将一次传送一个文本行，返回的不足值等于文本行的大小。
  • 读和写网络套接字（socket）：如果打开的文件对应于网络套接字，那么内部缓冲约束和较长的网络延迟会引起read和write返回不足值。对Unix管道调用read和write时，也有可能出现不足值。

实际上，除了EOF，在读和写磁盘文件时，不会遇到不足值。但是当处理网络套接字时，需要反复调用read和write函数来处理不足值，知道所有需要的字节都传送完毕。

RIO包

RIO包是一个自动为你处理不足值的I/O包，它建立在系统级I/O包之上。RIO包提供了两种不同类型的函数：

  • 无缓冲的输入输出函数：直接在存储器和文件之间传送数据，没有应用级缓存，它们对二进制数据读写到网络和从网络读写二进制数据尤其有用。
  • 带缓冲的输入函数：允许从文件中读取文本行和二进制数据，这些文件内容缓存在应用级缓冲区中，类似于printf这样的标准I/O函数提供的缓冲区。带缓冲的RIO输入函数是线程安全的。

共享文件机制

读取文件元数据

应用程序通过调用stat和fstat函数，检索到关于文件的信息（也称为文件的元数据(metadata)）。

stat函数以一个文件名作为输入，并填写下图所示的一个stat数据结构中的各个成员。fstat与stat相似，不过是以文件描述符而不是文件名为输入。

st_size成员包含了文件的字节数大小，st_mode成员则编码了文件访问许可位和文件类型。Unix识别大量不同的文件类型，普通文件包括某种类型的二进制或文本数据。对于内核而言，文件文件和二进制文件毫无区别。目录文件包含关于其他文件的信息。套接字是一种用来通过网络与其他进程通信的文件。Unix通过宏指令根据st_mode成员来确定文件的类型。如下图所示：

共享文件


除非你很清楚内核是如何表示打开的文件的，否则文件共享的概念非常难懂。内核用三个相关的数据结构来表示打开的文件。

  • 描述符表：每个进程都有它独立的描述符表，它的表项是由进程打开的文件描述符来索引的。每个打开的描述符表项指向文件表中的一个表项。
  • 文件表：打开文件的集合🈶一张文件表来表示的，所有的进程共享这张表。每个文件表的表项组成包括当前的文件位置、引用计数（即当前指向该表项的描述符表项数）、以及一个指向v-node表中对应表项的指针。关闭一个描述表会减少相应的文件表项中的引用计数。内核不会删除这个文件表表项，直到它的引用计数为零。
  • v-node表：桶文件表一样，所有的进程共享这张v-node表。每个表项包含stat结构中的大多数信息，包括st_mode和st_size。

上图中展示的是描述符1和4通过不同的打开文件表表项来引用两个不同的文件，这是没有共享文件，并且每个描述符对应一个不同的文件。后面的相关操作都是在该基础上进行的。

I/O重定向

Unix外壳提供了I/O重定向操作符，如:

unix > ls > bigming.txt

使得外壳加载和执行ls程序，将标准输出重定向到磁盘文件bigming.txt。I/O重定向使用dup2函数。

dup2函数拷贝描述符表项oldfd到描述符表项newfd，覆盖newfd以前的内容。如dup2(4, 1)之后如下图所示：

这里因为打开文件表中文件A的引用计数为0，所以被删除了，与之对应的v-node表项也因为没有文件表项指向它被删除了。而打开文件表中文件B的引用计数变为了2。

标准I/O

ANSI C定义了一组高级输入输出函数，称为标准I/O库，该库（libc）提供了关闭和打开文件的函数（fopen和fclose）、读和写字节的函数（fread和fwrite）、读和写字符串的函数（fgets和fputs），以及复杂的格式化的I/O函数（scanf和printf）。

标准I/O库将一个打开的文件模型化为一个流。一个流就是一个指向FILE类型的结构的指针。每个ANSI C程序开始时都有三个打开的流stdin、stdout和stderr，分别对应标准输入、标准输出和标准错误。

Unix I/O、标准I/O和RIO之间的关系如下图所示：

对于大多数应用程序来说，标准I/O更简单，是优于Unix I/O的选择。然而，因为对标准I/O和网络文件的一些相互不兼容的限制，使得Unix I/O比标准I/O更适用于网络应用程序。而RIO包则是针对读和写操作会出现不足值的情况通过反复读写操作，直到传送完所有的请求数据，自动处理不足值。
